# 🎙️ AI-Audio-Studio 

**AI-Audio-Studio** 是一款基于 Web 端的高性能人声分离与音频编辑工具。它深度集成了目前业内尖端的 AI 分离模型（如 **Roformer** 和 **Demucs v4**），并配合自研的 **“Mixer Studio (混音工作室)”**，为音乐制作人、DJ 和音频爱好者提供从“音轨拆解”到“创意重组”的一站式工作流。

---

## 1. ✨ 核心功能与技术亮点

### 1.1 全架构 AI 核心
* **多架构兼容**：支持 **MDXC (Roformer)**、**MDX-Net**、**VR Arch** 以及 **Demucs v4** 等主流架构。
* **BS-Roformer 强力驱动**：默认搭载 `BS-Roformer-Viper-1297` 模型，提供目前 SOTA 级的干净人声提取效果。
* **Demucs 四分轨**：使用 Demucs v4 模型可一键将音频拆分为 **人声 (Vocals)**、**鼓点 (Drums)**、**贝斯 (Bass)** 和 **其他 (Other)** 乐器轨道。

### 1.2 独立混音工作室 (Mixer Studio) 
* **独立功能入口**：在页面顶部设有独立混音工作室入口，不再局限于单一任务。
* **跨任务轨道叠加**：支持自由勾选历史任务中的任意音轨进行叠加合并。
* **无损合并引擎**：基于 `pydub` 库实现多音轨精准对齐与波形叠加，一键导出混音成品。

### 1.3 极致用户体验 (UX)
* **平滑虚拟进度条**：采用非线性减速算法模拟处理进度，确保进度条从 0% 到 100% 始终平稳移动，解决 AI 处理时的反馈真空。
* **唯一性命名规范**：遵循 `[任务ID]_[原名]_[音轨标签].mp3` 规则，支持中文名称且彻底杜绝浏览器预览重复的问题。
* **生产力套件**：支持一键 ZIP 打包下载处理结果，以及一键清理输出库功能。

---

## 2. 🛠️ 技术栈

* **后端**: Python 3.10+, Flask, Flask-CORS
* **AI 引擎**: `audio-separator` (ONNX Runtime GPU / PyTorch)
* **音频处理**: `pydub` (FFmpeg 支持)
* **前端**: HTML5, Tailwind CSS, JavaScript (SSE 异步通讯)

---

## 3. 📦 安装与配置

### 3.1 基础环境
* **确保您的系统安装了 Python 3.10 或更高版本，并拥有 NVIDIA GPU（建议 4GB 以上显存）。**

### 3.2 安装依赖库
* **在项目根目录下打开终端，执行以下命令安装必要组件：**
```bash
pip install -r requirements_gpu.txt
```
* **或cpu依赖**
```bash
pip install -r requirements_cpu.txt
````
### 3.3 配置 FFmpeg
* **混音功能高度依赖 FFmpeg 静态库。请确保：**

* **Windows: 下载 FFmpeg 并将 ffmpeg.exe 放置在 项目根目录/bin/ffmpeg/ 文件夹下。**

* **自动检测: 程序启动时会自动将该路径注入系统环境变量。**

### 3.4 模型准备
* **系统启动后会自动检测并下载模型至 models/ 文件夹。若您的网络环境受限，请根据 config.py 中的模型 ID 手动下载权重文件。**

## 4. 🚀 使用方法
### 4.1 启动后端
* **在 PyCharm 或终端中运行以下命令开启服务：**
```bash
python app_gpu.py
```
### 4.2 访问界面
* `在浏览器中访问本地服务地址：http://127.0.0.1:5001。`

### 4.3 处理流程
* **上传音频：直接拖拽或点击上传一个或多个音频文件。**

* **选择模型：在卡片列表中选择需要的 AI 架构（如选择 Demucs 开启四分轨）。**

* **开始分离：点击“开始处理”，等待进度条 100% 后显示结果预览。**

* **混音与导出：**

* **点击页面顶部导航栏的 “🎚️ 混音工作室” 按钮。**

* **在弹窗中勾选想要合并的不同音轨。**

* **点击“开始导出”，完成后即可直接试听或下载混音成品。**

## 5. 📁 目录结构说明
```bash
AI-Audio-Studio/
├── app_gpu.py            # 后端 Flask 主程序
├── config.py             # 路径配置与模型列表映射
├── bin/ffmpeg/           # FFmpeg 执行文件存放目录
├── models/               # AI 权重文件存放区 (.onnx / .ckpt)
├── uploads/              # 临时上传区 (处理后自动销毁)
├── outputs/              # 最终分离结果存放区
└── templates/
    └── index.html        # 前端交互响应式页面
```
### ⚠️ 开发者建议
* **显存溢出预防：若显存小于 4GB 导致处理报错，请在 config.py 或界面中调低 overlap 参数。**

* **文件存储清理：频繁使用后请点击页面上的“清理”按钮，防止结果文件占用过多磁盘空间。**

* **任务排队：系统内置了任务锁（Semaphore），会自动对多个文件进行排队处理，以保护显卡不被过载。**
